## 实施内容

### 1. 重构 `run_agent.py`
将原有的简单入口脚本重构为完整的 Hermes 工作流编排器。

#### 核心函数：`async def run(goal, start_url, task_guidance)`

**三阶段工作流**：

### 阶段 1：意图检索 (Retrieval)
```python
# 检索相关技能
relevant_skills = skill_manager.search_skills_by_intent(
    query=goal,
    top_k=3
)

if relevant_skills:
    # 找到相关技能
    matched_skill = relevant_skills[0]
    task_guidance = f"【相关经验】\n{matched_skill['guidance_sop']}"
    print(f"🧠 触发肌肉记忆！匹配到已有技能: {matched_skill['intent_description']}")
else:
    # 未找到相关技能
    print("💭 未找到相关经验，将进行全新探索")
```

**功能**：
- 调用 `SkillManager.search_skills_by_intent()` 检索本地技能库
- 如果匹配到技能，提取 `guidance_sop` 作为 `task_guidance`
- 打印醒目日志，显示匹配的技能信息
- 如果未匹配到，`task_guidance` 为空，进行全新探索

### 阶段 2：执行任务 (Execution)
```python
result = await workflow.run_task(
    goal=goal,
    start_url=start_url,
    task_guidance=task_guidance or ""
)
```

**功能**：
- 将任务交给 `AgentWorkflow` 执行
- 如果有 `task_guidance`，注入到 workflow 的系统提示词中
- 返回 `TaskResult`，包含执行结果和操作历史

### 阶段 3：自我反思与学习 (Reflection & Evolution)
```python
if result.success:
    # 情况 A：全新任务探索成功
    if matched_skill is None:
        # 反思提炼
        sop_text = await skill_manager.reflect_and_extract(
            goal=goal,
            action_history=result.actions_executed,
            llm_client=llm_client
        )
        
        # 保存新技能
        new_skill = skill_manager.save_new_skill(
            goal=goal,
            sop_text=sop_text
        )
        
        print(f"✨ 新技能已习得并写入神经中枢！")
    
    # 情况 B：已有技能复用成功
    else:
        skill_manager.increment_success_count(matched_skill['skill_id'])
        print(f"🎯 已有技能复用成功，增加技能权重")

# 情况 C：任务失败
else:
    print("💔 任务失败，不记录为技能")
```

**功能**：
- **情况 A（全新任务探索成功）**：
  - 调用 `reflect_and_extract()` 进行反思提炼
  - 调用 `save_new_skill()` 保存新技能
  - 打印新技能信息
- **情况 B（已有技能复用成功）**：
  - 调用 `increment_success_count()` 增加技能权重
  - 打印更新后的成功次数
- **情况 C（任务失败）**：
  - 不记录为技能
  - 打印失败日志

### 2. 主入口函数：`async def main()`

```python
async def main():
    """主入口函数"""
    # 初始化日志
    setup_logger()
    
    # 测试任务
    goal = "在淘宝搜索 RTX 5090 并告诉我第一项的价格"

        await run(

```

**功能**：
- 初始化日志系统
- 设置测试任务
- 调用 `run()` 执行 Hermes 工作流

## 架构约束遵守情况

### ✅ 最高层业务编排
- `run_agent.py` 作为最高层的业务编排脚本
- 不修改任何底层模块（`workflow.py`、`skill_manager.py`）的内部逻辑

### ✅ 单向数据流
- 检索 → 执行 → 反思，单向流动
- 不产生逆向数据回写
- 各阶段独立，互不干扰

### ✅ 清晰的职责划分
- **run_agent.py**：工作流编排
- **SkillManager**：技能管理
- **AgentWorkflow**：任务执行
- **LLMClient**：反思推理

### ✅ 完整的异常处理
- KeyboardInterrupt 处理（用户中断）
- Exception 处理（执行出错）
- 反思提炼失败的容错处理

## 核心设计决策

### 1. 三阶段工作流
- **检索 (Retrieval)**：利用已有经验
- **执行 (Execution)**：完成任务目标
- **反思 (Reflection)**：提炼新经验

### 2. 技能匹配策略
- 使用 `search_skills_by_intent()` 进行关键词匹配
- 返回 Top-3 相关技能
- 使用最相关的技能（第一个）

### 3. 反思触发条件
- **全新探索成功**：反思提炼并保存新技能
- **技能复用成功**：增加技能权重
- **任务失败**：不记录为技能

### 4. 用户体验优化
- 醒目的日志输出（使用 emoji 和分隔线）
- 清晰的阶段划分
- 详细的技能信息展示
- 最终总结报告

## 实际效果预期

## 性能指标

- **时间复杂度**：O(n*m)，n 为技能数量，m 为查询词数量
- **优化空间**：使用向量数据库可优化到 O(log n)

### 反思性能
- **LLM 调用**：1 次（反思提炼）
- **时间开销**：约 5-10 秒（取决于 LLM 响应速度）

### 存储性能
